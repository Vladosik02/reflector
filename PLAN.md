# План действий: сервис поиска двойников

## Цель
Собрать монолитный одностраничный сайт на Next.js (TypeScript + Tailwind), который служит **скелетом** для последующего наложения дизайна. Светлая премиальная гамма, русскоязычный интерфейс, три логические секции на одной странице.

---

## Технический стек

| Слой | Решение | Зачем |
|---|---|---|
| Фреймворк | Next.js 14 (App Router) | Готов к full-stack, SSR, API routes для будущего загрузки фото |
| Язык | TypeScript | Типобезопасность, понятный код для делегации |
| Стили | Tailwind CSS | Быстрый layout, дизайн-токены в одном файле |
| Шрифт | Inter (через `next/font`) | Универсальный sans-serif для премиум-UI |
| Иконки | Inline SVG | Без зависимостей, легко переопределить |
| Состояние | Локальное (`useState`) | Скелет — без global state |
| Линтер | (опционально) eslint-config-next | Стандарт |

## Файловая структура

```
doppelganger-app/
├── PLAN.md                   ← вы здесь
├── RESEARCH.md               ← исследование и палитра
├── README.md                 ← как запустить и доработать
├── package.json
├── next.config.mjs
├── tsconfig.json
├── tailwind.config.ts
├── postcss.config.mjs
├── .gitignore
├── app/
│   ├── layout.tsx           ← глобальный layout + шрифт
│   ├── page.tsx             ← главная (импорт секций)
│   └── globals.css          ← Tailwind base + дизайн-токены
├── components/
│   ├── Header.tsx
│   ├── Hero.tsx
│   ├── HowItWorks.tsx
│   ├── Pricing.tsx
│   ├── UploadSection.tsx
│   └── Footer.tsx
├── lib/
│   └── content.ts           ← все тексты и тарифы (легко править)
└── public/
    └── .gitkeep
```

**Принципы**
- Все тексты вынесены в `lib/content.ts` — переводчик/копирайтер правит один файл.
- Цвета — в `tailwind.config.ts` как именованные токены (`brand-bg`, `brand-accent`, ...).
- Секции — независимые компоненты, можно перекидывать местами.

## Спецификация «лимитированных фото» (для тарифа Max)

В тарифе Max обещаны «уникальные источники, доступные ограниченное время». На уровне UI это:
- Бейдж `Доступно 24:00:00` (обратный отсчёт) на карточках совпадений из таких источников.
- Янтарный акцент `#B5651D`, тонкая граница.
- Карточка-обёртка в pricing-блоке поясняет, что это значит.

На уровне данных (для бэкенд-разработчика):
- Источник имеет поле `expiresAt: ISO timestamp`.
- Фронт показывает таймер; по истечении — фото blur + надпись «Срок истёк».
